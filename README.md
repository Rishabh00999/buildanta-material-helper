# Buildanta Material Helper

A single-page Next.js app for first-time homeowners in Kanpur. Type a
construction material (e.g. "TMT bar", "cement", "tiles") and get 3 things
to check before buying it, generated by the OpenAI API.

Built for the Buildanta Full-Stack Developer (AI-Focused) pre-screen task.

## How it works

- **Frontend** (`src/app/page.tsx`): one input field + submit button. Calls
  `POST /api/material-check` and renders the result, an empty-input notice,
  or an error message depending on what comes back.
- **Backend** (`src/app/api/material-check/route.ts`): a Next.js Route
  Handler that validates the input server-side (never trusts the client
  alone), calls the OpenAI Chat Completions API, and returns either
  `{ result }` or `{ error }` with an appropriate HTTP status. All OpenAI
  calls are wrapped in try/catch so a failure returns a clean JSON error
  instead of crashing the route.

## Local setup

```bash
npm install
cp .env.example .env.local   # then paste your real OpenAI key into .env.local
npm run dev
```

Open http://localhost:3000.

### Required environment variable

| Variable | Where to get it |
|---|---|
| `OPENAI_API_KEY` | https://platform.openai.com/api-keys |

`.env.local` is git-ignored — never commit real keys.

## Deploying on Vercel

1. Push this repo to GitHub (public).
2. Import the repo on [vercel.com/new](https://vercel.com/new).
3. In the project's **Settings → Environment Variables**, add
   `OPENAI_API_KEY` with your real key.
4. Deploy. Vercel auto-detects Next.js — no extra build config needed.

## Error handling covered

- **Empty input** — checked both client-side (before any network call) and
  server-side (the API route rejects empty/whitespace-only input with a
  400 and a friendly message).
- **API failure** — OpenAI errors (auth, rate limit, network, malformed
  response) are caught and turned into a clean error message with the
  right status code (`429` for rate limits, `502` for upstream failures,
  `500` for missing server config). The UI shows the message without
  crashing.
- **Oversized input** — capped at 80 characters server-side as a basic
  guard against prompt abuse.
